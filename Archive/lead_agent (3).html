Overview

Reframing AI design from
interface thinking to behavior systems

This self-initiated project explored a core question: what does it mean to design an AI agent from the inside out? Instead of starting with UI flows or conversation scripts, the system was designed beginning with values, behavior rules, trust boundaries, and escalation logic.

The Problem

Traditional lead automation systems optimize for conversion efficiency. This project explored whether an AI agent could instead optimize for trust, emotional awareness, and human collaboration while still supporting go-to-market objectives.

Core Challenge

Design a lead intelligence agent capable of qualifying prospects, tracking emotional states across conversations, evaluating product fit honestly, and escalating to humans at trust-critical moments.

Design Shift

The project revealed that AI product design is shifting from surface-level interaction design toward behavioral architecture — defining what an agent believes, how it behaves under pressure, and when it should defer to humans.

Architecture

Six architectural iterations
toward a trust-aware agent system

The system evolved from a simple intent-driven chatbot into a multi-layer collaboration architecture with emotional state modeling, dual behavior layers, and human-AI coordination modes.

01

Soul.md — Value Constitution

Instead of defining workflows first, the project began with a behavioral constitution describing the agent's values, conflict hierarchy, and escalation philosophy. This became the source of truth for all subsequent architecture and testing decisions.

02

Psychological Trust State Machine

The architecture shifted from intent → response logic toward emotional state tracking. The system models 8 trust states, including hesitation, objection, evaluation, and commitment, allowing the agent to adapt its posture based on relationship context instead of isolated prompts.

03

Dual Behavior Layer

Customer-facing and rep-facing behaviors were separated into distinct engines. Customers receive trust-paced conversational guidance, while reps receive concise operational briefings, alerts, and actionable summaries.

Reflection

The designer's role is shifting
from interface designer to value architect

This project reframed AI UX as behavioral system design rather than conversational interface design. The most important deliverables were not screens, but value hierarchies, trust models, escalation logic, and behavioral QA frameworks. As AI systems become more autonomous, the designer's responsibility increasingly moves upstream — shaping how systems behave under ambiguity, emotion, and trust-sensitive conditions.
